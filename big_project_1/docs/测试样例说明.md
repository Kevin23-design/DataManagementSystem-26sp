# 问卷系统测试样例说明

## 1. 文档目的
本文档用于满足课程项目中的测试要求，提供可直接执行和记录的测试样例。

覆盖范围包含以下六类核心测试：
1. 创建问卷测试
2. 添加题目测试
3. 跳转逻辑测试
4. 校验测试
5. 提交问卷测试
6. 统计测试

测试形式包含：
1. 自动测试
2. 手工测试
3. API 测试

## 2. 测试环境与前置条件

### 2.1 环境信息
1. 操作系统：Windows
2. Node.js：18+
3. MongoDB：6.0+
4. 后端地址：http://localhost:3000

### 2.2 前置条件
1. MongoDB 已启动并可连接。
2. 后端服务已启动。
3. 测试账号已准备：
	1. 创建者账号 A（用于创建问卷、查看统计）
	2. 普通账号 B（用于填写、权限校验）
4. 测试数据库为独立数据集，避免污染生产数据。

## 3. 执行方式说明

### 3.1 自动测试
1. 测试文件：server/tests/survey-system.e2e.test.js
2. 执行命令：在 server 目录运行 npm test
3. 输出结果：Jest 报告，显示通过/失败用例数

### 3.2 API 测试
1. 使用工具：Postman、Apifox 或 curl
2. 核心验证：HTTP 状态码、返回字段、错误信息
3. 记录方式：保留请求体、响应体截图或导出报告

### 3.3 手工测试
1. 在浏览器进行登录、建问卷、填写、查看统计操作。
2. 核对页面行为与后端响应是否一致。
3. 对异常提示文案进行截图记录。

## 4. 公共测试数据设计
所有样例默认使用以下问卷模板（可按实际接口返回 ID 替换）：

1. 问卷：测试问卷-基础流程
2. 题目：
	1. Q1 单选题，必答，选项 A/B
	2. Q2 多选题，必答，选项 苹果/香蕉/西瓜，minSelect=2, maxSelect=3
	3. Q3 文本题，必答，minLength=2, maxLength=50
	4. Q4 数字题，必答，min=0, max=120, integerOnly=true
3. 跳转规则：
	1. Q1= A 跳转到 Q3
	2. Q1= B 按默认顺序到 Q2

## 5. 测试用例清单

### TC-01 创建问卷测试（API + 自动）
1. 测试目标：验证登录用户可成功创建问卷。
2. 测试步骤：
	1. 使用账号 A 登录获取 token。
	2. 调用创建问卷接口，传入标题、说明、匿名开关。
3. 输入：
	1. title=课程反馈
	2. description=用于测试
	3. allowAnonymous=true
4. 预期输出：
	1. 状态码 201
	2. 返回字段包含 _id、shareCode、status=draft
5. 实际输出：
	1. 自动化测试项“创建问卷成功（已登录）”通过（196 ms）。
	2. 自动化测试项“创建问卷失败（未登录）”通过（17 ms），未登录返回 401。
6. 结果：通过

### TC-02 添加题目测试（API + 自动）
1. 测试目标：验证可在草稿问卷中新增题目。
2. 测试步骤：
	1. 对草稿问卷依次添加单选、多选、文本、数字题。
	2. 查询题目列表确认数量与类型。
3. 输入：题型与校验参数见公共测试数据。
4. 预期输出：
	1. 每次新增状态码 201
	2. 题目列表总数为 4，类型顺序正确
5. 实际输出：
	1. 自动化测试项“可添加四类题型并正确保存”通过（218 ms）。
	2. 自动化测试项“发布后不可添加题目”通过（171 ms）。
6. 结果：通过

### TC-03 跳转逻辑测试（手工 + API + 自动）
1. 测试目标：验证题目跳转按规则执行。
2. 测试步骤：
	1. 发布问卷。
	2. 填写 Q1 选择 A。
	3. 继续作答并提交。
	4. 查询保存结果中的 questionOrder。
3. 输入：Q1=A，Q3 和 Q4 按规则作答。
4. 预期输出：
	1. 路径跳过 Q2
	2. 提交成功，保存顺序包含 1、3、4（若 4 在路径中）
5. 实际输出：
	1. 自动化测试项“命中单选跳转规则时跳过中间题”通过（193 ms）。
	2. 自动化测试项“不命中跳转规则时按默认顺序走下一题”通过（182 ms）。
6. 结果：通过

### TC-04 校验测试（API + 自动）
1. 测试目标：验证必答和约束规则生效。
2. 测试步骤：
	1. 提交必答题为空。
	2. 提交多选数量不足。
	3. 提交数字超出范围。
3. 输入：
	1. Q1 空
	2. Q2 仅选 1 项
	3. Q4=121
4. 预期输出：
	1. 状态码 400
	2. 错误信息与题目规则匹配
5. 实际输出：
	1. 自动化测试项“必答题为空时提交失败”通过（149 ms），返回 400。
	2. 自动化测试项“多选数量与数字范围校验生效”通过（156 ms），校验错误被正确拦截。
6. 结果：通过

### TC-05 提交问卷测试（API + 自动 + 手工）
1. 测试目标：验证合法填写可提交并落库。
2. 测试步骤：
	1. 按规则完成全部必答题。
	2. 调用提交接口。
	3. 在 responses 中检查新增记录。
3. 输入：合法答案集。
4. 预期输出：
	1. 状态码 201
	2. 返回 responseId
	3. 数据库新增 1 条提交记录
5. 实际输出：
	1. 自动化测试项“匿名问卷可匿名提交，且可多次提交”通过（172 ms）。
	2. 自动化测试项“非匿名问卷未登录提交失败，登录后成功”通过（249 ms）。
6. 结果：通过

### TC-06 统计测试（API + 自动 + 手工）
1. 测试目标：验证统计结果计算正确。
2. 测试步骤：
	1. 连续提交 2-3 份不同回答。
	2. 调用整卷统计接口。
	3. 核对单选计数、多选计数、文本列表和数字均值。
3. 输入：
	1. 样本 1：单选 A，数字 80
	2. 样本 2：单选 B，数字 60
4. 预期输出：
	1. totalResponses=2
	2. 单选 A=1，B=1
	3. 数字 average=70，min=60，max=80
5. 实际输出：
	1. 自动化测试项“统计接口返回单选计数与数字聚合结果”通过（204 ms）。
	2. 自动化测试项“非创建者无权限查看统计”通过（238 ms）。
6. 结果：通过
